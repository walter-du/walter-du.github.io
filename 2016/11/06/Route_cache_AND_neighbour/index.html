route cache 中 dst->neighbour 和 dst->hh 何时分配和赋值 ?

dst->neighbour 是在第一次匹配路由后 create route cache 时分配和初始化的;

dst->hh 是在发包前发送 arp 完整交互以后分配和赋值的;

PS:

  • 明确了 neigh 就确定了 xmit ;
  • Linux 3.5 之后取消 route cache ;
  • Add static arp:
    ip neigh add 8.8.8.8 lladdr 00:01:02:03:04:05 dev eth1 nud perm
